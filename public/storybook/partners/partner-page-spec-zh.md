# 合作夥伴介面 — 公開頁面 + 私人工作區

每位合作夥伴在 markluce.ai 上有**兩個明確分開的介面**。這個規則適用於 Audrey、Arita 以及未來所有合作夥伴。

## 1. 公開頁面 — `app.markluce.ai/{slug}`
對外公開。目的：介紹合作夥伴的目標市場與廣告文案。這是合作夥伴的**門市**。

### URL 規則
- `app.markluce.ai/{partner-slug}` — 例如 `/audrey`、`/arita`。
- slug 獨一無二、小寫、`[a-z0-9-]`。

### 公開 CMS 欄位（合作夥伴可編輯，全世界可見）
- **主標題**（繁中／英／日 三語）。
- **目標市場描述** — 這位夥伴服務誰。
- **廣告文案／訴求** — 2–3 句價值主張，用夥伴自己的語氣。
- **品牌色** — 單一主色 hex。
- **Logo／頭像** — 選擇性上傳。
- **示範作品** — 公開展示清單（縮圖 + 標題 + 示範繪本連結）。
- **CTA 按鈕** — 標籤 + 動作（登入、加入 LINE 群、開始問卷）。
- **LINE 群連結** — 合作夥伴自己的群組 URL。
- **每本／每分級建議零售價** — 合作夥伴在平台建議區間內選擇具體數字。在公開頁顯示為「建議零售價 / Suggested list price」。詳見 [`../pricing/list-price-zh.md`](../pricing/list-price-zh.md)。
- **折扣提示** — 自由文字說明合作夥伴通路內可能有折扣，例如「加入 LINE 群取得會員報價」。

### 公開頁面上平台控制的部分
- URL、路由、授權、基礎設施。
- 共用模板清單（夥伴分支；不能改基礎模板）。
- 頁尾：「Powered by markluce.ai」。

## 2. 私人工作區 — `app.markluce.ai/{slug}/admin`
**只有該合作夥伴自己**（與 Mark 作為全域管理者）可見。目的：管理客戶、繪本、白名單、定價、備註。這是合作夥伴的**後台**。

### 私人欄位
- **客戶清單** — 終端使用者檔案（LINE user ID、顯示名稱、標籤、家長聯絡）。每位夥伴自己持有，其他夥伴絕對看不到。
- **每位孩子的備註** — Arita 通路用：家長諮詢紀錄、學習目標、什麼有效／無效。
- **繪本庫存** — 這位夥伴生成的每一本書，含 metadata、PWA 狀態、白名單、撤銷控制。
- **PWA 白名單** — 哪些 LINE user ID 可以安裝哪些書。每本獨立可編輯。
- **每位客戶實際成交價** — 合作夥伴選擇性記錄對每位終端使用者實際收的金額（平台外交易）。私人分析比較實際價 vs 建議零售價、顯示折扣分布。Mark 看不到。
- **私人分析** — 每本書、每位客戶、每月使用量。
- **家長諮詢問卷回覆** — Arita 通路用：家長回傳的表單。

### 存取控制
- 合作夥伴以 LINE 登入（同一個登入）。
- 合作夥伴角色解鎖自己 slug 的 `/admin`。
- **嚴格隔離：** 一位夥伴不能看到另一位夥伴的工作區。
- Mark 擁有跨所有 slug 的全域管理權。

## 歸因
- LINE Login 重導帶入 `state={slug}`。
- 首次接觸的合作夥伴 slug 永久記在使用者紀錄上。
- 所有生成與計費事件都標記來源合作夥伴。

## 未來所有合作夥伴
未來新加入的合作夥伴（Audrey + Arita 之後）都適用同樣的雙介面架構：
- 透過 CMS 編輯自己的公開頁 — 目標市場 + 廣告文案對外可見。
- 在自己的私人工作區管理客戶 — 與其他夥伴隔離。
- 每加一位夥伴不用寫新程式碼。全是設定。
