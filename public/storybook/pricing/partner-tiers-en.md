# Partner tiers (draft)

**Fee model:** per-generation infra fee, with tiers layered on top. Higher tier = better unit rate + more benefits. See [`partner-terms.md`](partner-terms.md).

Exact numbers TBD after first two partners validate the model — structure below is locked.

## Starter (default entry)
- Hand-held onboarding with Mark.
- Single partner slug.
- Base template inventory (questionnaires, layouts, voices, art styles).
- Azure TTS (zh-TW / en / ja-JP).
- gpt-image-1 illustrations.
- Manual monthly invoicing (usage × Starter unit rate).
- **Baseline per-generation rate** (highest of the three tiers).
- Shared support channel.

## Growth
Everything in Starter, plus:
- **Lower per-generation unit rate** (volume discount).
- Partner CMS full access (multi-author).
- Usage analytics dashboard.
- Priority support channel.

## Pro
Everything in Growth, plus:
- **Lowest per-generation unit rate.**
- **Premium voices** — ElevenLabs v3 unlocked as an end-user upsell.
- **Custom character consistency** — Flux + per-character LoRA training for signature characters.
- White-labeled partner page options (subdomain / custom domain — M2+).
- Direct-with-Mark support.

## Add-ons (any tier)
- Additional slugs (for partners running multiple brands).
- EPUB3 export (M2 feature).
- Custom template commissioning by Mark.

## Pricing principles
- **B2B only** — see [`partner-terms.md`](partner-terms.md). No revenue share. No end-user billing.
- **One meter:** per-generation infra fee. Tier determines the unit rate and which features are unlocked.
- Price engine costs + margin; don't try to capture end-user value directly.
- Starter has no lock-in — partners can stay there forever if their volume is low. Higher tiers only make sense once volume / features justify the upgrade.
- Be generous with early partners (Audrey, Arita) to validate the model; tighten later.
