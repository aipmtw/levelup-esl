# 未來網域 / 授權概念

## 背景
markluce.ai 是 **Mark 的第一個生意**。這個網域永遠是 Mark 的 — 是旗艦品牌。但**商業模式本身**（引擎 + 合作夥伴模式 + PWA 交付 + 繁中優先兒童繪本）在設計上是可以**轉售／授權**給其他人，以不同品牌網域經營的。

未來要保留的兩種可能性：

1. **授權平台** 給另一位經營者，讓對方用自己的品牌網域運作。Mark 繼續經營 markluce.ai，收授權費 + 分潤。
2. **分拆子品牌** 到一個全新的網域，專攻特定市場（例如日本優先、全球英語、或高階出版品牌），而 markluce.ai 仍然是台灣／繁中旗艦。

## 為什麼現在就要想
- 這讓程式、模板系統、合作夥伴 CMS 在底層保持**品牌無關**。平台不應該把 "markluce.ai" 這種字串寫死 — 所有品牌元素都應該透過合作夥伴／租戶設定流入。
- 讓每本書的 PWA 命名機制從一開始就**與品牌解耦**（PWA 的顯示名稱由合作夥伴控制，不是由平台控制）。
- 讓 Mark 在被問到「這個能買嗎？」時，可以回答「可以，這是價目表」。

## 未來轉售／子品牌的候選網域名稱

以下名稱**不是用來取代 markluce.ai**。它們是**第二品牌的選項**，如果 Mark 授權或分拆時才會啟用。

### 兒童繪本直接訴求（感性、消費者友善）
- **tinytales.ai** — 溫暖、好記、品牌中性、全球通用。
- **kidlit.ai** — 兒童文學圈內用語，短而精。
- **storyseed.ai** — 「種下點子，長成書本」的生成器隱喻。
- **ourbook.ai** — 個人化／家庭訴求（「我們的」＝親子）。
- **lullaby.ai** — 睡前故事定位（可能已被註冊，需確認）。
- **yomibook.ai** — "yomi" 在日文是「閱讀」；用於鎖定日本市場的子品牌。
- **bambubook.ai** — 竹子 = 亞洲意象，適用於繁中 + 日文市場。

### B2B 引擎／平台定位
- **bookforge.ai** — 貼合「合作夥伴在此引擎上打造」的定位。工廠／熔爐意象。
- **storyforge.ai** — 同樣意象，偏重故事。
- **storypress.ai** — 出版／印刷意象。
- **bookmint.ai** / **pagemint.ai** — 每個請求「鑄造」一本個人化書。
- **booklab.ai** — 實驗室 = 創作者工具。
- **storycraft.ai** — 工藝取向，比 forge 柔和。

## 建議 — 三個入選名單

1. **tinytales.ai** — 最強的對家長直接訴求品牌；未來若 Mark 要自己推第二品牌，最適合消費端露出。
2. **bookforge.ai** — 最乾淨的 B2B 平台名；精準貼合「合作夥伴在此引擎上打造」的概念。Mark 要把引擎授權給其他經營者時最合適。
3. **yomibook.ai** — 保留給日本市場分拆使用 — ja-JP 為基礎 + en，正好對應 markluce.ai 用 zh-TW 為基礎 + en 的結構。

## Mark 的行動項目
- 查 `tinytales.ai`、`bookforge.ai`、`yomibook.ai` 的註冊狀態。
- 若有合理價格的可註冊，先註下來 — 便宜的保險。
- 程式、品牌、文案都保持**參數化**，讓換膚到以上任一網域是設定問題，不是重寫問題。
