# 概念 — 中文（繁體）

## 主要目標市場

**0–12 歲兒童。** 鎖定繁體中文家長市場 — 台灣優先，其次香港與海外華僑，日本視為鄰接市場。

**語言支援：**
- **繁體中文（zh-TW / zh-Hant） — 基礎語言，恆常包含。必須是繁體，禁止簡體。**
- **英文（en） — Mark 主力產品的必備加購項。**
- **日文（ja-JP） — 選購加購項。**

> **硬性規則：** 所有中文接觸點 — UI、TTS 語音、LLM 輸出、生成繪本文字 — 都必須是**繁體中文**。簡體中文不在範圍內。LLM prompt 必須明確指示 "Traditional Chinese, zh-TW, Taiwan locale"。TTS 只使用 `zh-TW-*` 語音。

## 平台（markluce.ai，Mark 擁有）

markluce.ai 是 **Mark 的第一個生意**，這個網域保留在 Mark 名下。商業模式本身 — 引擎 + 合作夥伴模式 — 設計上可以在未來以**另一個品牌網域授權或轉售**。詳見 [`future-domain-zh.md`](future-domain-zh.md)。

- 一個**基礎服務平台**，產生內容讓合作夥伴去賣：
  - **兒童專屬繪本** — 多頁插畫故事書，0–12 歲，繁體中文為基礎，可加購英文與／或日文。
  - **基礎模板** — 問卷模板、版型、頁數／年齡分級、朗讀聲音、插畫風格 — 讓每位合作夥伴在此基礎上打造自己的風格。
- **共用基礎設施：** LINE Login、Azure Neural TTS、圖像生成、LLM 故事寫作、書本渲染、PWA 與離線下載包輸出。
- **合作夥伴頁面**：`app.markluce.ai/{partner-slug}`，每位夥伴自己擁有頁面文案、風格、示範作品、LINE 群組連結。
- Mark **不做 B2C**，不向終端使用者收小額費用，只向合作夥伴收費。

## 目前三位合作夥伴（都在主目標內，通路不同）

三位主合作夥伴都銷售 0–12 歲兒童繪本，繁體中文為基礎 + 英文 + 選購日文。差別在於**如何接觸終端使用者**以及專精的年齡區段。

### Audrey — 職場家長／同儕社群通路
- **背景：** Audrey 是一位全職專業工作者，**自己有兩個孩子都在 0–12 歲範圍內**。她有自己的**家長同儕社交圈**。
- **通路：** 透過她的家長社交圈口耳相傳，加上她自己的 LINE 群。家長對家長、人對人的信任。
- **產品切入點：** 先替自己的孩子客製繪本，再擴及朋友的孩子，然後更廣。

### Arita — 老師／一對一與家長討論通路
- **背景：** Arita 是一位英語老師，**已經有自己的 0–12 歲學生**。
- **通路：** 她為每一位特定學生設計個人化的教材，**與家長討論後**再製作。老師策展、家長把關。
- **產品切入點：** 每本繪本都是客製的教材 — 針對該位學生的程度、興趣，以及與家長商定的學習目標。比 Audrey 的同儕通路更深度接觸、每本書感知價值更高。

（Arita 舊的 `markluce.ai/arita` 成人英語學習者工具保留為實驗性次要介面，**不是** M1 主要焦點。M1 聚焦在 Arita 的兒童教學產品。）

### Amy — 新手家長／0–3 專精通路
- **背景：** Amy 是 **Mark 的女兒**，有一個 **1 歲的兒子**（Mark 的外孫）。她本來就是實體個人化童書的付費用戶，親身熟悉這個品類。
- **通路：** 她的新手家長社交圈 — 首胎媽媽、baby shower 朋友、0–3 幼兒的家長。人對人、信任密度高。她每天用產品在自己兒子身上，朋友親眼看到實際使用。
- **年齡專精：** 0–3 分級 — 這是 Audrey 與 Arita 較不著墨的區間。Amy 補上覆蓋缺口。
- **家庭治理：** 父女關係以明確護欄處理 — 見 [`../partners/family-governance-zh.md`](../partners/family-governance-zh.md)。Amy 的計費與待遇與 Audrey、Arita 完全相同。

三位主合作夥伴都用這個產品發展自己的社交圈，所有合作夥伴的款項都在**平台外**結算 — Mark 開立發票，合作夥伴以銀行轉帳／LINE Pay／現金支付。平台不做線上金流。見 [`../pricing/partner-terms-zh.md`](../pricing/partner-terms-zh.md)。

### 合作夥伴年齡分級涵蓋矩陣 — 無斷層、無重疊

每位合作夥伴行銷的對象是她**親身正在經歷**的年齡帶，以及符合她**人生階段**的社交圈。這對可信度極度關鍵：家長是跟他們視為同儕的人買東西，不是跟假裝懂他們的人買。

| 年齡分級 | Amy（新手家長，1 歲兒） | Audrey（職場家長，6 歲與 10 歲） | Arita（兒童英語老師） |
|---|---|---|---|
| **0–3** | ✅ **主力** — 她兒子就在這裡 | ❌ 已過此階段 | ~ 老師角度，優先度低 |
| **4–6** | ❌ | ✅ **主力** — 她 6 歲兒在這裡 | ✅ 核心教學年齡 |
| **7–9** | ❌ | ✅ 橋接兩個孩子的年齡 | ✅ 核心教學年齡 |
| **10–12** | ❌ | ✅ **主力** — 她 10 歲兒在這裡 | ✅ 上層教學年齡 |

**社會定位／人生階段：**
- **Amy** 賣給其他處於嬰幼兒階段的**新手家長**。她在嬰兒群組、baby shower 聊天室、首胎家長 LINE 群裡活躍。人對人的信任最強，因為她就是跟客戶處於同一階段的人。
- **Audrey** 賣給其他**國小學齡孩子的全職職場家長**。她在學校家長 LINE 群、放學接送聊天圈、職場媽媽同儕網絡裡活躍。時間緊、收入穩、重視雙語教育。
- **Arita** 賣給**已經聘請她當兒童英語老師的家長**。她的關係是專業而溫暖（老師對家長）、一對一客製諮詢、每本書單價較高，因為每本都是客製教材。

三位沒有互相競爭。Amy 的新手媽媽朋友永遠不會是 Audrey 的客戶。Audrey 6 歲兒同學的家長永遠不會是 Amy 的客戶。Arita 的學生永遠不會是另外兩位的客戶 — 因為學生的家長已經挑了 Arita 當老師。這個分隔誠實又自然。

## 離線 — PWA 優先（必備）

每本生成的繪本都以 **Progressive Web App（PWA）** 形式交付，每本擁有獨一無二的名稱，並**綁定指定的 LINE 帳號**才能使用。使用情境：家長先在有網路時載入書本，再把裝置交給小孩在**飛航模式**（飛機、車上、無 wifi）下播放。

- **首次啟動（需網路）：** 必須 LINE 登入，驗證合作夥伴白名單，資源與 auth token 快取到本機。
- **後續啟動（離線可用）：** 以本機快取的 LINE token 驗證，書本完全離線播放 — 文字、圖片、朗讀音訊全來自 service worker 快取。
- **夥伴控制：** 合作夥伴把 PWA 指派給一個或多個 LINE 帳號（家長的帳號）。只有這些帳號能安裝／播放。
- 次要輸出：`.zip` 包（PDF + 圖片 + MP3 + `meta.json`），給需要再分發或列印的夥伴用。

詳見 [`../tech/pwa-offline-zh.md`](../tech/pwa-offline-zh.md) 與 [`../product/offline-bundle-spec-zh.md`](../product/offline-bundle-spec-zh.md)。

## 使用流程

1. 家長進入合作夥伴頁面 `app.markluce.ai/audrey`。
2. 以 LINE Login 登入（歸因透過 `state={slug}` 帶入）。
3. 回答簡短問卷（孩子名字、年齡、興趣、目標、語言組合、風格）。
4. 平台產生文字（符合年齡、繁中為基礎 + 選擇的語言）+ 插畫（主角一致）+ 朗讀音訊（Azure Neural TTS）。
5. 家長拿到一個**個人化 PWA** — 安裝到手機／平板，綁定 LINE 帳號，可在飛航模式離線播放。
6. 可選擇下載 `.zip` 原始素材包，用於列印或再分發。
