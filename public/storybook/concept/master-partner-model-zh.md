# 主合作夥伴模式 — 雙層合作架構

## 核心想法
Mark 把自己的**直接計費關係維持在少數幾位主合作夥伴** — 起步是**三位**：Audrey、Arita、Amy。永遠（或至少很長時間）。她們成為**主合作夥伴（master partner）**。她們每個人都可以招募、輔導並把平台轉租給自己社交網內的**子合作夥伴（sub-partner）**，讓子合作夥伴用自己的社交網經營相同的兒童繪本 PWA 生意，但掛在 Audrey 或 Arita 的旗下。

Mark 的客戶名單維持小巧。收入透過 Audrey 和 Arita 的子網擴展。

## 這對 Mark 為什麼好
- **支援範圍維持在 2 人。** Mark 只回覆 Audrey 和 Arita 的訊息。子合作夥伴由主合作夥伴輔導，不找 Mark。
- **品質控管外包給主合作夥伴。** Audrey 和 Arita 才是握有真實客戶關係的人 — 她們最清楚誰適合經營這個產品。不適任的子合作夥伴 = 主合作夥伴自己處理。
- **誘因對齊。** Audrey 和 Arita 按自己談的條件從子網賺錢。Mark 不介入。
- **Mark 的收入自然擴展。** 按次計費的對象是主合作夥伴，含**所有**子合作夥伴的使用量。主合作夥伴的子網長大，自然升級到更高等級。
- **沒有 MLM 的不良觀感** — 前提是**嚴格維持雙層**。沒有子子孫孫。Mark 不付推薦佣金。真實價值 = 真實繪本賣給真實家長。

## 這對 Audrey 和 Arita 為什麼好
- 她們自己也成為**小型平台經營者**，不只是內容創作者。
- 既有的社交網（Audrey 的家長圈、Arita 的老師圈）變成招募志同道合子合作夥伴的土壤。
- 她們自己決定怎麼跟子合作夥伴分潤 — Mark 不規定。
- 她們建立護城河：子網依附於她們，不是直接依附於 Mark。

## 這對子合作夥伴為什麼好
- **進入門檻低。** 不用跟 Mark 談條件，直接掛在既有主合作夥伴旗下。
- **真實的輔導。** 主合作夥伴做過、知道什麼有效、手把手帶。
- **共用模板與經驗。** 主合作夥伴可以分享自己驗證過的繪本模板、問卷變體、定價實驗。

## 架構 — 嚴格雙層

```
Mark（平台擁有者）
  │
  ├── 主合作夥伴：Audrey   （4–12，職場家長通路）
  │     ├── 子合作夥伴：例如「學校家長好友 Mei」
  │     ├── 子合作夥伴：例如「幼兒園老師 Jenny」
  │     └── 子合作夥伴：...
  │
  ├── 主合作夥伴：Arita    （4–12，兒童老師通路）
  │     ├── 子合作夥伴：例如「另一位英文老師 Lin」
  │     └── 子合作夥伴：...
  │
  └── 主合作夥伴：Amy      （0–3，新手家長通路 — Mark 的女兒，見 family-governance）
        ├── 子合作夥伴：例如「她 baby-group 的朋友」
        └── 子合作夥伴：...
```

**只有兩層。** 子合作夥伴不能再招募自己的子子合作夥伴。如果一位子合作夥伴成長到超過主合作夥伴的輔導範圍（罕見、需刻意），可以經 Mark 審核後**升級為主合作夥伴** — 那是升遷，不是新層級。

## Mark 提供給主合作夥伴的功能
- **子合作夥伴管理** — 在主合作夥伴的私人工作區中建立子 slug、邀請子合作夥伴、撤銷。
- **巢狀歸因** — LINE Login `state` 帶入 `{master_slug}/{sub_slug}`，計費時彙總。
- **子合作夥伴 CMS** — 子合作夥伴在主合作夥伴的命名空間下編輯自己的公開頁，範圍由主合作夥伴設定。
- **模板繼承** — 子合作夥伴從主合作夥伴的客製模板分支（不只是 Mark 的基礎模板）。
- **使用量彙總** — 主合作夥伴的計費儀表板顯示子合作夥伴的使用量；Mark 向主合作夥伴請款彙總金額。
- **輔導套件** — Mark 給 Audrey 和 Arita 一本現成的輔導手冊（如何 onboarding 子合作夥伴、如何拆分潤、如何處理支援問題）。

## 命名空間

### 主合作夥伴公開頁
`app.markluce.ai/audrey` — Audrey 自己的門市。她的目標市場、她的示範繪本、她的建議零售價、她的 LINE 群。

### 子合作夥伴公開頁（兩種選項，選一）
- **選項 A — 路徑式：** `app.markluce.ai/audrey/mei` — 一看就是「Mei，由 Audrey 帶領」。
- **選項 B — 扁平 slug 加父層 metadata：** `app.markluce.ai/mei` 頁尾有「Audrey 的子網之一」。

**建議：選項 A（路徑式）。** 維持層級可見、強化主合作夥伴的品牌、歸因變得簡單、升級時 URL 遷移乾淨（`/audrey/mei` → `/mei`）。

## 計費
- **Mark 只向主合作夥伴計費。** 收費結構與今日相同（按次基礎設施費 + 分級）。
- **生成彙總：** 子合作夥伴的生成計入主合作夥伴的等級配額與單價。
- **主合作夥伴對子合作夥伴的計費在平台外進行。** Audrey 決定怎麼向 Mei 收費 — 分潤、固定費、免費、隨便她。Mark 不碰那筆錢。
- **爭議回退：** Mark 保留在主合作夥伴消失或行為不當時直接支援子合作夥伴的權利。罕見的升級路徑。

## 對等級的影響
主合作夥伴能力是一個**功能門檻**，加在 Growth 與 Pro 等級。Starter 等級不能招募子合作夥伴。

| 等級 | 主合作夥伴能力 | 典型使用 |
|---|---|---|
| Starter | ❌ | 單兵經營者，沒有子合作夥伴。 |
| Growth | ✅ 最多 N 位子合作夥伴（例如 5 位） | 帶小型子網的主合作夥伴。 |
| Pro | ✅ 無上限子合作夥伴 | 完整主合作夥伴經營模式。 |

預期 Audrey 和 Arita 開始招募後會落在 **Pro**。

## 風險與緩解
| 風險 | 緩解 |
|---|---|
| MLM／老鼠會觀感 | 嚴格雙層。Mark 不付推薦佣金。真實產品 = 真實繪本。 |
| 主合作夥伴成為瓶頸／消失 | 升級路徑：Mark 可在緊急情況下直接支援子合作夥伴。子合作夥伴的客戶資料存在自己的私人工作區。 |
| 不良子合作夥伴損害主合作夥伴品牌 | 主合作夥伴在自己的工作區對子合作夥伴有撤銷權。 |
| 定價混亂（主與子都各自設建議零售價？） | 平台設建議區間。主與子都在區間內選。歸因有黏性，所以每位終端使用者只看到一個價格。 |
| 主與子之間互相蠶食 | 地理／利基劃分由主合作夥伴自己執行（Audrey 的子合作夥伴不應鎖定 Audrey 自己在攻的幼兒園）。Mark 不提供工具；主合作夥伴自行社交解決。 |

## 「在 PWA 裡賣語言教材」在這個模式下的意義
**價值單位**不變：客製化的多頁插畫兒童繪本，繁體中文為基礎（+ 英文、+ 選購日文），以 LINE 帳號綁定的 PWA 交付。詳見 [`../product/kid-book-spec-zh.md`](../product/kid-book-spec-zh.md)。

**通路**則倍增：Audrey 自己的家長圈、Arita 的老師—學生—家長三角、以及每位子合作夥伴各自獨立的社交網。同一個產品，更多出口。

## 升級路徑（罕見、需刻意）
一位子合作夥伴若：
- 在主合作夥伴旗下已建立可觀的獨立客戶群，
- 有扎實的紀錄（繪本品質、客戶滿意度、準時交付），
- 得到主合作夥伴的祝福，
- 經過 Mark 審核通話後核准，

可以**升級為主合作夥伴**。URL 從 `/audrey/mei` 遷移到 `/mei`。這預期很罕見 — 這個模式的整個重點就是讓 Mark 的直接客戶數維持很小。
