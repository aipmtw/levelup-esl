# 繪本生命週期 — 孩子成長中的重複購買

## 核心洞察
一個孩子不是一次性客戶。**一個孩子從 0 到 12 歲的整段歷程，價值是很多本繪本 — 不是一本。** 當孩子 1 歲時賣（或送）出第一本書的合作夥伴，可以在接下來 10 多年持續賣出符合孩子成長階段的繪本。

這把經濟模型從**一次性銷售**轉成**長期關係**。早期投資一位客戶的合作夥伴會隨著孩子長大持續收到回報。Mark 從該合作夥伴收到的每次生成費用也會隨時間安靜地複利。

Amy 的案例是最清楚的示範，但這個模式對 Audrey 和 Arita 也同樣適用。

## Amy 的客戶示意

取 Amy 任何一位新手家長朋友，叫她雅晨。雅晨有一個 1 歲的女兒。接下來 10 年 Amy 與雅晨的關係可能長這樣：

| 雅晨的孩子年齡 | Amy 生成、雅晨購買（或收到贈送） |
|---|---|
| **1 歲**（首次購買） | *認識自己* — 身體部位、家人、基本物件。繁中唯一。極簡。 |
| 1 歲 3 個月 | *我的一天* 作息本 — 早晨、吃飯、洗澡、睡覺。繁中唯一。 |
| 1 歲 6 個月 | *我喜歡的動物* — 搭配雅晨帶女兒去動物園的行程。繁中唯一。 |
| 1 歲 9 個月 | *雅晨寶貝的第一次生日* 紀念本 — 里程碑本，繁中，永久珍藏。 |
| **2 歲**（語言升級） | 同樣主題，但**加上簡單英文**與繁中並排。「狗狗 / Dog」。雙語接觸開始。 |
| **2 歲 6 個月**（主角首登場） | **主角故事** — 設計的故事，雅晨的女兒是主角。「小美的海邊冒險 / Mei's Beach Adventure」。雙語。 |
| 3 歲 | 新的主角故事，不同主題。字彙略廣。 |
| **4 歲**（分級轉換 0–3 → 4–6） | 第一本有真正情節的 10–12 頁繪本。雙語，句子更長，字彙更豐。 |
| 5 歲 | 有主題／寓意的主角故事（「分享」、「勇氣」）。 |
| 6 歲 | 入學前的第一本書。若雅晨家是多語環境，可加日文為第三語言。 |
| ... | ...延續 7–9 與 10–12 分級，每本更長、文字更豐、情節更精緻。 |

**10 年下來，雅晨可能從 Amy 這邊累積 15–25 本繪本。** 不是一本。

## 成長的三個軸

當合作夥伴為回頭客戶生成第 N 本書時，可以沿三個獨立軸向成長：

### 軸 1 — 主題／繪本類型
- **0–3 類型：** 認識自己、日常作息、感官（顏色／聲音）、動物、家人、里程碑紀念。
- **4–6 類型：** 主角冒險、主題探索（車輛、自然）、簡單情感故事（分享、勇氣）、短睡前故事。
- **7–9 類型：** 章節輕量主角冒險、主題深入（恐龍、太空、海洋）、角色驅動敘事。
- **10–12 類型：** 多章故事、類型小說（推理、奇幻）、字彙重點本、孩子興趣的非虛構書。

合作夥伴可以在類型間輪替，讓系列保持新鮮感 — 同一個孩子可能生日拿到里程碑本、暑假拿到主角冒險、開學拿到作息本。每本都感覺不同，但共享同一個主角。

### 軸 2 — **孩子作為主角**
核心升級路徑：從**一般主題本**（「顏色」、「動物」）升級到**孩子就是主角的故事**。

- **入門** — 「小美喜歡狗狗」（孩子被提到，主題是重點）。
- **中段** — 「小美去農場」（孩子出現在畫面，主題是背景）。
- **完整主角** — 「小美的龍冒險」（孩子推動情節、克服挑戰、是敘事中心）。

主角書**情感依附強得多** — 孩子認出自己會想反覆閱讀、家長當紀念品珍藏。能撐起更高的建議零售價，也會更快產生口碑擴散（「看，我小孩在這本書裡」）。合作夥伴應在 2 歲左右開始引入主角書，3 歲以後大量使用。

### 軸 3 — **語言隨時間分級升級**
孩子成長，書中的語言也隨之成長。同一位合作夥伴、同一個孩子、同一套引擎 — 在新的級別產生新書。

| 孩子年齡 | 典型語言組合 | 句子複雜度 |
|---|---|---|
| 0–1 | 只有繁中 | 每頁 1 個字或 1 個短語 |
| 1–2 | 只有繁中 | 每頁 1 個短句 |
| 2–3 | 繁中 + 基礎英文（配對標示） | 1 句 + 英文單字配對 |
| 3–4 | 繁中 + 英文（句子層級） | 每頁 1–2 句，兩種語言 |
| 4–5 | 繁中 + 英文（+ 選購日文） | 2–3 句，簡單對話 |
| 5–7 | 繁中 + 英文（+ 選購日文） | 短段落，情節弧線 |
| 7–9 | 繁中 + 英文（+ 選購日文） | 多段落頁，近似章節 |
| 9–12 | 繁中 + 英文（+ 選購日文） | 完整章節、字彙更豐、類型多元 |

**關鍵：** 分級升級是**在新的級別產生一本新書**，不是編輯舊書。1 歲的那本永遠是珍貴的典藏。2 歲的那本是家長書架上新出現的東西。系列成長，舊書永不丟棄。

## 平台如何支援

### 合作夥伴私人工作區中的每位孩子歷程
合作夥伴工作區中的每位終端使用者孩子都有一份檔案：
- 名字、年齡（隨時間自動增加）
- **繪本歷史** — 為這位孩子生成過的每一本書，含日期、分級、主題、語言組合、升級標記
- **下次建議** — 當孩子年齡進入新分級時，工作區提醒合作夥伴：「小美現在 4 歲。要不要為她產生第一本 4–6 分級繪本？根據她的歷史，建議主題：主角冒險（她的最愛）、動物（2 歲已有過，該是更豐富版本的時候了）。」
- **里程碑提醒** — 生日、開學、農曆新年等。觸發合作夥伴主動聯繫家長提出新的紀念本建議。

### 繪本類型分類
平台為每一本生成的繪本標記：
- `tier` — 0-3 / 4-6 / 7-9 / 10-12
- `type` — first-words / routine / sensory / animals / hero / milestone / themed / story / chapter / ...
- `theme` — 合作夥伴選擇的自由文字主題
- `languages` — zh-TW / zh-TW+en / zh-TW+en+ja
- `complexity_level` — 分級內 1–10 的複雜度尺度

合作夥伴用這些標記找出缺口（「這個孩子從沒有過主角本」），並規劃下一本。

### 升級標記
合作夥伴為回頭孩子產生新書時，平台根據孩子年齡 + 過往歷史自動建議適合的 `complexity_level`。合作夥伴隨時可覆寫。當 complexity_level 明顯跳級時，書在工作區被標記為**升級本** — 一個值得慶祝的進展事件，合作夥伴可以與家長一起慶祝（「您女兒升級了！這本有完整的英文句子」）。

## 對每個合作夥伴等級的收入意涵

- **Starter（無子合作夥伴、輕度使用）：** 合作夥伴圈內有 10 個終端孩子，每位每年 1–2 本，每年 10–20 本。低等級量、不錯的副收入。
- **Growth（小型網絡）：** 合作夥伴有 30 個終端孩子（自己的圈 + 幾位子合作夥伴的圈），每位每年 2–3 本，每年 60–90 本。有意義的穩定生意。
- **Pro（完整主合作夥伴模式含子合作夥伴）：** 像 Audrey 這樣的主合作夥伴，自己的圈 + 5 位子合作夥伴各自的圈，總共 150 位終端孩子，每位每年 2–3 本 → **每年 300–450 本**透過一位主合作夥伴等級下的平台流動。每次生成費用 × 400 本 = Mark 的真實收入，Audrey 的真實生意。

生命週期模式是 Pro 等級**值得存在**的原因。沒有重複購買，主合作夥伴的書本數會停滯；有了與孩子成長綁在一起的重複購買，即使合作夥伴不加新客戶，年年的量都會成長。

## 對 M1 的意義

M1 不需要完整的生命週期工具。M1 需要：
- **能為回頭孩子產生新書**，不用重新輸入所有資料（合作夥伴工作區中的每位孩子檔案）。
- **基本的繪本類型標記**（分級、主題、語言、複雜度）。
- **簡單的每位孩子繪本歷史檢視**（依日期排序的清單）。

M2 再加上建議引擎、里程碑提醒、升級標記、更豐富的目錄模板。

## 對三位合作夥伴的訊息

- **Amy：**「您 1 歲兒子的第一批書是一座圖書館的開始。每一年我會做新的、隨他一起長大的書。」從第一天就是終身關係。
- **Audrey：**「我已經為自己的 6 歲兒和 10 歲兒做過。每一年，新書、新的級別。您孩子的系列會一直長大。」以她自己兩條示範弧線建立可信度。
- **Arita：**「每個月我都會在課堂上見到您的孩子。每個月我都能做一本剛好對應我們這學期正在學的內容的繪本。」終極的循環教材。

三位合作夥伴賣的都是**關係**，不是單本繪本。繪本是交付物；關係才是產品。
