# Future domain / licensing concept

## Context
markluce.ai is **Mark's first business**. The domain stays Mark's forever — it's the flagship. But the **biz model itself** (engine + partner pattern + PWA delivery + Traditional-Chinese-first kid books) is designed to be **resaleable / licensable** under a different brand domain in the future.

Two future scenarios to keep open:

1. **License the platform** to another operator who runs it under their own brand domain. Mark keeps building markluce.ai and takes a license fee + revenue share.
2. **Spin out a sub-brand** under a fresh domain for a specific market (e.g. Japan-first, or global English, or a premium publisher brand) while markluce.ai stays as the Taiwan/zh-TW flagship.

## Why think about this now
- It keeps the code, template system, and partner CMS **brand-neutral underneath**. Nothing in the platform should hardcode "markluce.ai" strings — all branding flows through partner/instance config.
- It makes the offline PWA per-book naming scheme brand-agnostic from day one (the PWA's display name is partner-controlled, not platform-controlled).
- It lets Mark answer "can I buy this?" with a yes and a ready-made price sheet.

## Candidate names for a future resale / sub-brand domain

None of these replace markluce.ai. They are **options for a second brand** if Mark licenses or spins out.

### Kid-book direct (emotive, consumer-friendly)
- **tinytales.ai** — warm, memorable, partner-neutral, global appeal.
- **kidlit.ai** — insider term for children's literature, compact.
- **storyseed.ai** — conveys "plant the idea, we grow the book" generation metaphor.
- **ourbook.ai** — personalized / family angle ("our" = parent + kid).
- **lullaby.ai** — bedtime storybook positioning (likely taken; check).
- **yomibook.ai** — "yomi" = read in Japanese; useful if the sub-brand targets Japan.
- **bambubook.ai** — Asia/bamboo cue, works for zh-TW + ja markets.

### B2B engine / platform positioning
- **bookforge.ai** — matches the "engine partners build on" framing. Factory/forge metaphor.
- **storyforge.ai** — same metaphor, story-leaning.
- **storypress.ai** — publishing/press metaphor.
- **bookmint.ai** / **pagemint.ai** — minting personalized books per request.
- **booklab.ai** — labs = tooling for makers.
- **storycraft.ai** — craft-oriented, slightly softer than forge.

## Recommendation — three to shortlist

1. **tinytales.ai** — strongest direct-to-parent brand; works as a consumer-facing sub-brand if Mark ever spins up a Mark-owned second brand.
2. **bookforge.ai** — cleanest B2B platform name; exactly matches "engine partners build on". Best if Mark licenses the engine to another operator.
3. **yomibook.ai** — reserved for a Japan-market spin-out that pairs ja-JP first with en, mirroring how markluce.ai pairs zh-TW first with en.

## Action items for Mark
- Check availability of the three shortlisted domains (`tinytales.ai`, `bookforge.ai`, `yomibook.ai`).
- If any are available at a reasonable price, park them now. Cheap insurance.
- Keep all code / branding / copy **parameterized** so that re-skinning to any of these domains is a config change, not a rewrite.
