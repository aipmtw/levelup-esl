# MarkLuce.ai 大規模改版紀錄
**日期：** 2026-04-14
**工具：** Claude Code (Opus 4.6, 1M context)
**操作者：** Mark 陳炳陵（馬克路思科技）

---

## 一次對話完成的工作量

在一次 Claude Code 對話中，從零開始完成以下所有項目，包含規劃、開發、部署、驗證。

---

## 1. /storybook/ — AI 個人化語言繪本商業計畫

- 從外部目錄 `biz-plan/` 讀取 72 份 Markdown 文件（繁中＋英文雙語）
- 建立 tabbed SPA 頁面，8 大分頁：總覽、概念、產品、技術、合作夥伴、定價、路線圖、市場
- 每個分頁有子導航按鈕，動態載入對應 Markdown 並以 Marked.js 渲染
- 支援中英文即時切換

## 2. /scottsu/ — Scott Su 競選網站（含 AI Chatbot）

- 從外部專案複製前端 SPA + 知識庫 JSON
- 修正所有圖片路徑為相對路徑
- 建立 3 個 Vercel Serverless Functions：
  - `/scottsu/api/chat` → Azure OpenAI (GPT-4o) 代理，含 `buildSystemPrompt()` 從知識庫動態建構系統提示
  - `/scottsu/api/tts` → Azure Speech Services TTS 代理
  - `/scottsu/api/info` → 回傳知識庫 JSON
- 從 Azure CLI 取得 5 組環境變數（AZURE_OPENAI_ENDPOINT 等）

## 3. /agents/ — AI 智慧體目錄（212 個 AI 專家角色）

- 分析 GitHub 上游 repo `jnMetaCode/agency-agents-zh`（MIT 授權）
- 發現本地版本（175 個）已過時，上游已有 199 個
- 建立新 GitHub repo `aipmtw/ai-agents-directory`
- 從上游提取 37 個新增 agent，逐一翻譯為繁體中文
- 建構完整 standalone HTML viewer（212 agents, 18 部門）
- README 明確標註上游來源與授權
- 首頁卡片從外部連結改為內部 `/agents/`

## 4. /admin/ — 管理後台

- 從 git 歷史還原舊版 admin 頁面
- 擴展為 4 分頁：Overview、ESL Analytics、SQL Runner、Quick Links
- SQL Runner 含預設按鈕（一鍵 ALTER TABLE、查看 schema 等）
- Quick Links 整合 Supabase Dashboard（兩個專案）、Vercel、GitHub、Azure Portal
- 建立 `/api/sql` Serverless Function（admin 密碼保護）

## 5. Features 文章系統升級

### Clean URL
- 從 `/features/?slug=xxx` 改為 `/features/2026-04-slug`
- 日期前綴自動從文章的 `date` 欄位提取
- Vercel rewrite 設定 SPA fallback
- 前端改用 `location.pathname` 解析

### 參考來源（Wikipedia 風格）
- Supabase `features` 資料表新增 `sources` jsonb 欄位
- 4 篇文章全部更新原始來源（共 16 筆參考資料）
- 前端渲染為 Wikipedia 格式：`[1] 標題. domain. [日期].`
- 文末顯示文章 metadata + 永久連結

## 6. Supabase 全權管理

- 取得 `sb_secret_` service key
- 存入 `.env`（已 gitignore）
- 後續可直接執行 DDL/DML，不需手動操作 Dashboard

## 7. Google Search Console

- 新增驗證 meta tag 並通過驗證
- 更新首頁 meta description（從通用描述改為具體產品列表）
- 提交 Request Indexing 更新 Google 快取

## 8. 首頁更新

- 新增 3 張卡片：Scott Su 競選網站、AI 個人化語言繪本、AI Agents 目錄
- AI Agents 目錄從外部連結改為內部頁面
- 更新數據：212 個 → 18 部門

## 給 AIA 學校的回饋

這次改版展示了 **AI 協作開發的實際生產力**：

1. **一次對話 = 一個完整 sprint**：從需求分析、架構設計、編碼、測試、部署到 SEO，全部在一次 Claude Code 對話中完成。
2. **跨系統整合**：同時操作 GitHub、Supabase、Azure、Vercel、Google Search Console，AI 能串接所有工具鏈。
3. **真實產品，非 demo**：所有成果已上線運作，不是投影片上的截圖。
4. **Human-in-the-loop 很重要**：AI 提方案，人做決策（URL 格式、Ref 風格、資料庫 key 授權）。最好的結果來自持續的人機對話，而非一次性的指令。
